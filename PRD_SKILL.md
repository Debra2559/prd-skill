# PRD 写作 Skill

## 适用场景
写需求文档（PRD）、多语言文案 key 表、埋点表时使用。

---

## 一、PRD 文档规范

### 文档结构（固定顺序）
- 标题格式：`【产品线】功能名称`
- 一、更新日志
- 二、背景
- 三、需求详情
- 四、埋点（飞书链接）
- 五、多语言（飞书链接）

### 字体
微软雅黑，不用 Arial。

---

### 一、更新日志

表格三列：**日期 / 变更内容 / 变更人**

- 变更人格式：`@姓名-职级`，如 `@金腾辉`
- 内容按时间**倒序**排列
- 条目类型标注：
  - `【排期】`：列出待确认事项 + 对应负责人
  - `【产品走查】`：记录走查发现，附飞书链接
  - `【新建】`：首次创建

---

### 二、背景

只写**两条** bullet，不多写：

- **大背景**：现状是什么、有什么问题、为什么成本高
- **本次需求要解决的问题**：在现有 XX 基础上，新增 XX 能力，达到什么效果

背景后可附竞品对比截图或表格（可选）。

---

### 三、需求详情

#### 3.1 需求目标
- 3-5 条 bullet
- 每条一句话，格式：`功能点 + 达到什么效果`
- 不写废话，直接说做什么

#### 3.2 需求详情
- 先写 UI 稿链接
- 主体是**详情表**，三列：

| 页面 / 组件 | 原型 | 逻辑 & 交互 |
|---|---|---|

**逻辑 & 交互列写法：**
- 用 `【加粗标题】` 分块，如【弹窗头部】【底部按钮】【关闭方式】
- 每块内用 bullet 展开，**重点词加粗**
- 边界情况、危险操作、状态变化要单独列出
- 口语化，简练，不堆砌形容词
- 危险操作要有视觉强调说明（如：标题文字红色、按钮变红）

#### 3.3 需求清单（可选，复杂需求使用）

表格三列：**模块 / 需求 / 备注**

- 按模块分组
- 每条需求一句话，描述清楚做什么
- 备注列写优先级或特殊说明

---

### 四、埋点 / 五、多语言

只放飞书链接，不在文档内展开：

```
[表格名称](飞书链接)
```

---

## 二、多语言文案 Key 表规范

### 表格格式（飞书）

五列：**页面 / 字段说明 / 中文话术 / 截图 / 对应 key**

### Key 命名规范

```
{origin}_{模块}_{组件}_{字段}
```

- 全小写，下划线分隔
- **origin**：产品线前缀，默认 `skywork`，可按需调整
- 示例：`{origin}_scheduled_task_channel_title`

### 其他规则

- 动态内容用占位符 `{变量名}`，如：
  - 「已执行{n}次」
  - 「选择你要联系{XXX}的方式（XXX 要替换成用户配置的 bot 名字）」
- 不需要多语言的内容注明：「无需多语言翻译」
- 动态数据（如群聊名称、用户昵称）不需要多语言，不列入表格
- 截图列留空，由负责人自己补
- 按页面/模块分组，组名单独一行标注，如：`【创建定时任务-手动】`

---

## 三、埋点表规范

### 表格格式（飞书）

八列：**模块 / 事件类型 / 截图 / event（唯一）/ origin / entity / event / 事件特殊参数 key / 备注**

### Event 唯一值命名规范

```
{origin}_{entity}_{event}
```

- 全小写，下划线分隔
- **origin**：产品线前缀，默认 `skywork`，可按需调整
- **entity**：操作对象，如 `scheduled_task_page`、`im_entry`
- **event**：动作类型，见下方枚举

### 事件类型枚举

| 事件类型 | event 值 |
|---|---|
| 曝光 | `show` |
| 点击 | `click` |
| 确认 | `confirm` |
| 关闭 | `close` |

### 常用事件特殊参数

| 参数名 | 说明 | 示例值 |
|---|---|---|
| `tab` | 当前 tab | `task_list / exec_history` |
| `source` | 来源页面 | `chat_card / task_detail` |
| `im_type` | IM 平台类型 | `telegram / slack / feishu / whatsapp / discord` |
| `im_status` | 连接状态 | `0`（失败）/ `1`（成功）|

### 其他规则

- 参数格式：`参数名: 可选值1 / 可选值2`
- 备注列：补充枚举值的中文解释，如：`chat_card 对话流小卡 / task_list 任务列表`
- 按模块分组，模块名单独一行加粗标注
- 截图列留空，由负责人自己补

---

## 四、调整 Origin 前缀

如需更换产品线前缀，只需修改本 skill 中所有 `skywork` 为新前缀，其他规范不变。
